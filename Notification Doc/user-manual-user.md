คู่มือผู้ใช้งาน — การแจ้งเตือน (Notifications)

ภาพรวมสั้น ๆ
- ผู้ใช้จะได้รับการแจ้งเตือนเมื่อมีการเปลี่ยนสถานะของรายงาน (Incident) ที่เกี่ยวข้องกับตนเอง เช่น ผู้รายงาน หรือผู้ที่เกี่ยวข้อง
- การแจ้งเตือนแสดงเป็นรายการใน "กระดิ่ง" บนมุมขวาบน และสามารถเข้าไปดูรายการทั้งหมดที่หน้า /notifications

สิ่งที่ผู้ใช้ทำได้
- ดูรายการแจ้งเตือน (หน้า dropdown และหน้าเต็ม)
- คลิกเพื่อเปิดรายละเอียด (ลิงก์จะไปที่ /incidents/{id})
- มาร์กว่าอ่านแล้ว (ปุ่ม "Mark read")
- ลบการแจ้งเตือนได้ถ้าต้องการ

ตัวอย่างการใช้งาน
1. ผู้ใช้ A รายงานปัญหา → สถานะ: PENDING
2. แอดมินเปลี่ยนสถานะเป็น REVIEWED → ผู้ใช้ A ได้รับ notification ว่า "รายงาน: <title> - REVIEWED"
3. ผู้ใช้เปิดการแจ้งเตือนเพื่อดูรายละเอียดของรายงาน

ข้อควรทราบ
- ถ้าไม่เห็นแจ้งเตือน ให้รีเฟรชหน้าเว็บหรือเช็คว่าคุณล็อกอินอยู่ด้วย token ถูกต้อง
- การแจ้งเตือนอาจจะถูกสร้างแต่ยังไม่แสดงถ้า API สำหรับดึงแจ้งเตือนยังไม่ถูก implement (Task 2)

คำแนะนำแก้ปัญหาทั่วไป
- "ไม่มีการแจ้งเตือน": ตรวจสอบ GET /api/notifications ว่าคืนค่าข้อมูลหรือไม่
- "คลิกแล้วไม่ไปหน้า": ตรวจสอบว่า `/incidents/{id}` มีหน้า detail (Task 5)

ติดต่อฝ่ายช่วยเหลือ
- หากพบบั๊กหรือข้อความผิดพลาด ให้แนบตัวอย่าง notification JSON (จาก API) และเวลาเกิดเหตุ เพื่อช่วยทีมแก้ไขเร็วขึ้น
